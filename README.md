# Автозаказ Про — прототип v2

Мини-проект для локального запуска:

- **web** — простой интерфейс на Next.js
- **api** — backend на NestJS
- **python-service** — расчёт автозаказа и генерация Excel

## Что умеет прототип

- принимает **только iiko XLSX**
- проверяет продажи по **штрихкодам**
- использует **встроенных тестовых поставщиков**
- сначала показывает **предпросмотр автозаказа на экране**
- отдельно показывает **товары с неизвестным поставщиком / неизвестным штрихкодом**
- формирует **Excel-документ заказа для каждого поставщика** по шаблону `Выгрузка заказа.xlsx`
- даёт скачать готовые документы

## Архитектура

```text
apps/web         -> интерфейс
apps/api         -> NestJS API
services/python  -> Python-алгоритм
shared-data      -> встроенные поставщики, шаблоны, примеры
```

## Важный файл с логикой расчёта

Формула автозаказа находится здесь:

`services/python/autozakaz/calculator.py`

Если потом нужно сохранить 1-в-1 вашу старую логику, менять надо **только этот файл**.  
В текущей версии используется базовая формула:

```text
средние продажи в день = продано за период / число дней в периоде
заказ = ceil(((средние продажи в день * coverage_days * sales_multiplier) + reserve_units) / pack_size) * pack_size
если заказ > 0 и заказ < min_order_qty, то заказ = min_order_qty
```

## Быстрый запуск

### 1) Требования

- Node.js 20+
- npm 10+
- Python 3.11+
- pip

### 2) Установить зависимости

В корне проекта:

```bash
npm install
python -m pip install -r services/python/requirements.txt
```

### 3) Переменные окружения

#### apps/api/.env

```env
PORT=4000
WEB_ORIGIN=http://localhost:3000
PYTHON_BIN=python3
PYTHON_SCRIPT=services/python/calculate_autozakaz.py
STORAGE_DIR=storage
SUPPLIERS_FILE=shared-data/suppliers.json
ORDER_TEMPLATE=shared-data/templates/Выгрузка заказа.xlsx
```

#### apps/web/.env.local

```env
NEXT_PUBLIC_API_URL=http://localhost:4000
```

### 4) Запуск backend

```bash
npm run dev:api
```

### 5) Запуск frontend

В новом терминале:

```bash
npm run dev:web
```

### 6) Открыть приложение

```text
http://localhost:3000
```

## Как пользоваться

1. Открыть веб-интерфейс
2. Загрузить iiko-файл `.xlsx`
3. При необходимости дважды нажать на квадрат в правом верхнем углу — откроются настройки
4. Нажать кнопку расчёта
5. На экране появится:
   - сводка по автозаказу
   - список товаров по поставщикам
   - неизвестные штрихкоды / товары
   - кнопки скачивания Excel по каждому поставщику

## Что лежит в shared-data

- `suppliers.json` — встроенные тестовые поставщики
- `templates/Выгрузка заказа.xlsx` — шаблон заказа поставщику
- `examples/Автозаказ_Про_iiko_тест_продаж_март_2026_v2.xlsx` — пример входного iiko-файла
- `catalogs/` — исходные тестовые каталоги поставщиков

## Ограничения текущего прототипа

- только формат **iiko**
- поставщики пока встроены в программу
- файловая история хранится в папке `storage/`
- без БД
- без авторизации
- без реального фонового брокера задач
